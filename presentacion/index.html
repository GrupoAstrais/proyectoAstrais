Descripción
Una herramienta de productividad con motivación incorporada.

Astraïs permite gestionar tareas, hábitos y objetivos, pero los conecta con XP, Ludiones, logros, cosméticos, grupos y minijuegos para que cada avance tenga una consecuencia visible.

Problema

Las apps tradicionales suelen abandonarse por monotonía, falta de recompensa inmediata y poca sensación de progreso.

Justificación

La gamificación aporta objetivos claros, feedback rápido y recompensas que ayudan a sostener la constancia en el tiempo.

Valor diferencial

El proyecto une utilidad real con una experiencia más dinámica: productividad, economía virtual, socialización y progreso personal en un mismo producto.

override suspend fun performBasicRegister(req): Boolean {
    val user = dao.getUsuario(req.email)

    if (user == null) {
        val hash = hashPassword(req.passwd)
        val uid = dao.createUser(
            req.name, req.email, hash, req.lang)
        dao.createGroup(uid, req.name, "", true)

        val code = (100000..999999).random()
        dao.saveConfirmationCode(uid, code.toString())
        sendEmail(req.email, code.toString())
        return true
    }
    return user.esta_confirmado == 0
}

fun generateAccessToken(user): String {
    return JWT.create()
        .withSubject(user.id.toString())
        .withExpiresAt(Date(System.currentTimeMillis()
            + 10 * 60 * 1000))  // 10 min
        .sign(Algorithm.HMAC256(accessSecret))
}

fun generateRefreshToken(user): String {
    return JWT.create()
        .withSubject(user.id.toString())
        .withExpiresAt(Date(System.currentTimeMillis()
            + 7 * 24 * 60 * 60 * 1000))  // 7 días
        .sign(Algorithm.HMAC256(refreshSecret))
}

private fun generateInviteCode(): String {

        .encodeToString(bytes)  // 32 chars

post("/groups/joinByCode") {
    val uid = call.principal<JWTPrincipal>()!!
                         .subject?.toInt()
    val data = call.receive<JoinByCodeRequest>()

    val ret = getGroupRepoImpl()
        .joinByCode(uid, data.code)
    when (ret) {
        OK -> call.respond(HttpStatusCode.OK)
        INVALID_CODE -> call.respond(BadRequest)
        EXPIRED -> call.respond(BadRequest)
        REVOKED -> call.respond(BadRequest)
        MAX_USES_REACHED -> call.respond(BadRequest)
        ALREADYJOINED -> call.respond(BadRequest)
    }
}
Validación 
override suspend fun joinByCode(requesterId, code) {
    val hash = sha256(code)
    val invite = getDao().getGroupInviteByHash(hash)
        ?: return INVALID_CODE

    if (invite.revokedAt != null)
        return REVOKED
    if (invite.expiresAt <= now)
        return EXPIRED
    if (invite.usesCount >= invite.maxUses)
        return MAX_USES

    getDao().tryConsumeInvite(invite.id, now)
    getDao().addUserToGroup(requesterId, invite.gid)
    return OK
}

Cuatro validaciones: ¿existe? ¿revocado? ¿expirado? ¿usos agotados? Solo si pasa todo se une al grupo.

Calcular recompensas 
fun calcularXp(tipo, prioridad, frecuencia): Int {
    val multPrio = 1.0 + (prioridad * 0.5)
    val multFreq = getFrecuencyMultiplier(frecuencia)

    val xpBase = when (tipo) {
        HABITO -> 20
        UNICO -> 50
        OBJETIVO -> 400
    }
    return (xpBase * multPrio * multFreq)
        .toInt()
}

Sumar al usuario 
override suspend fun completeTask(tid, uid): Boolean {
    val tarea = getDao().getTaskById(tid)
    val usuario = getDao().getUserById(uid)

    // Suma XP y Ludiones
    usuario.xp_total += tarea.recompensa_xp
    usuario.xp_actual += tarea.recompensa_xp
    usuario.ludions += tarea.recompensa_ludion

    // Si es hábito, actualiza racha
    if (tarea.tipo == HABITO) {
        updateStreak(tarea)
    }
    return true
}

Validación de compra 
override fun buyCosmetic(uid, cid): BuyCosmeticResponse {
    val user = Users.findById(uid)
        ?: return USER_NOT_FOUND
    val cosmetic = Cosmetic.findById(cid)
        ?: return COSMETIC_NOT_FOUND

    if (user.ludions < cosmetic.price)
        return INSUFICIENT_CURRENCY
    if (Inventory.exists(uid, cid))
        return ALREADY_HAS_OBJECT

    user.ludions -= cosmetic.price
    Inventory.new(uid, cid)
    return OKAY
}

La tienda valida saldo suficiente y que no tengas el objeto antes de descontar Ludiones.

fun buyItem(id: Int, onSuccess: () -> Unit) {
    if (sessionManager.isGuest()) return
    viewModelScope.launch {
        _state.update { it.copy(isLoading = true) }
        try {
            repository.buyCosmetic(id)

            onSuccess()
            _uiEvent.value = StoreEvent.BuySuccess

                e.message ?: "Fondos insuficientes")
        }
    }
}

Servidor

1
Problemas encontrados con OAuth
OAuth nativo en Android
En Android no fue posible reutilizar directamente el flujo OAuth del servidor.
Para poder autenticarse correctamente fue necesario implementar un flujo OAuth nativo en la aplicación móvil, ya que en el servidor se abre una página web de forma automatica y era peor usar (WebView / navegador externo) para completar el login.
Problemas con el callback web
Durante el callback OAuth en web aparecieron problemas porque el servidor no dispone de un dominio accesible desde el proveedor OAuth.
Esto impide llamar al callback de (redirect URI) y completar la autenticación. Curiosamente, si tienes el servidor en la misma maquina, si funciona con localhost.
