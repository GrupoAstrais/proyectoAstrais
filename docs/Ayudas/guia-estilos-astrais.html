Guia de estilos Astrais

Documento unico para alinear la interfaz de astrais-web y AstraisAndroid. Define tokens, superficies, componentes y reglas de uso compatibles con CSS/Tailwind y Jetpack Compose Material 3.

Principios Tokens Color Tipografia Layout Componentes Plataformas Accesibilidad

01

Principios visuales

Astrais usa una estetica oscura, espacial y funcional. La marca debe sentirse ludica, pero las pantallas de productividad siguen siendo densas, legibles y rapidas de escanear.

Oscuro por defecto

El fondo base es casi negro. Las superficies elevadas usan transparencias y gradientes tenues, nunca blanco puro como base.

Acentos del tema

Los colores primario, secundario y terciario se aplican a acciones, estados, iconos y paneles segun el tema equipado.

Glassmorphism controlado

Usar blur, borde claro y sombras suaves. La transparencia debe mejorar jerarquia, no reducir lectura.

Reglas de color

  • Primario: acciones principales, tarjetas importantes, seleccion, XP.
  • Secundario: calendario, informacion, navegacion auxiliar, estados activos frios.
  • Terciario: exito, progreso, habitos, recompensas positivas.
  • Error: fallos, borrado, validaciones y estados destructivos.
  • Texto: siempre sobre contraste suficiente; usar versiones `muted` solo para metadatos.

Panel primario comun

Superficies importantes usan una mezcla tenue del primario: fondo oscuro 88%, primario 30%, fondo alternativo con secundario 14%. 

.astrais-primary-panel-bg {
  background: linear-gradient(
    160deg,
    color-mix(in srgb, var(--astrais-background) 88%, var(--astrais-background-alt) 12%),
    color-mix(in srgb, var(--astrais-primary) 30%, transparent),
    color-mix(in srgb, var(--astrais-background-alt) 86%, var(--astrais-secondary) 14%)
  );
}

Reglas de layout

  • Desktop web: paneles en grids densos con altura controlada y scroll interno sin mostrar scrollbar cuando sea util.
  • Android: columnas verticales con `LazyColumn`, bottom bar fija y tarjetas con padding de 16-20dp.
  • Las pantallas de trabajo no usan heroes de marketing; empiezan directamente con la funcionalidad.
  • Todo contenedor con formato fijo debe tener `min-height`, `aspect-ratio` o tracks de grid estables.

07

Implementacion por plataforma

La guia no sustituye a los archivos fuente. Define el contrato para mantener ambos clientes sincronizados.

Web

  • Fuente de verdad: astrais-web/src/styles/colors.css.
  • Temas dinamicos: applyThemeColors actualiza variables CSS en :root.
  • Componentes importantes: usar astrais-primary-panel-bg o variantes con `color-mix`.
  • Fondos globales: wallpaper orbital fijo detras de la app; las pantallas no deben taparlo con imagenes opacas.
root.style.setProperty('--astrais-primary', theme.primary)
root.style.setProperty('--astrais-secondary', theme.secondary)
root.style.setProperty('--astrais-tertiary', theme.tertiary)

Android

  • Fuente de verdad: ui/theme/Color.kt, Theme.kt, Type.kt, Shapes.kt.
  • Temas dinamicos: AstraisandroidTheme(userTheme) mapea backend a Material 3.
  • Superficies: usar AstraisGlassSurface, AstraisGlassCard y MaterialTheme.shapes.
  • Tarjetas: fondo con alpha entre 0.04 y 0.12 para glassmorphism; no usar superficies solidas salvo modal critico.
darkColorScheme(
  primary = prim,
  secondary = sec,
  tertiary = tert,
  background = bg,
  surface = surf
)

08

Accesibilidad y consistencia

El tema es oscuro y translúcido, asi que el contraste debe revisarse en cada componente que mezcle colores.

Hacer
  • Usar texto principal para informacion critica y `muted` solo en metadatos.
  • Reservar `Press Start 2P` para titulos breves; el cuerpo debe ser legible.
  • Mantener borde visible en tarjetas sobre fondos animados.
  • Usar estados hover/focus claros en web y feedback tactil en Android.
  • Usar iconos con texto cuando la accion no sea obvia.
Evitar
  • No colocar texto largo con tipografia pixel.
  • No usar paneles completamente opacos salvo dialogos o alertas.
  • No cambiar semantica de color entre plataformas.
  • No depender solo del color para error, completado o seleccion.
  • No mezclar radios sin escala; usar 6, 10, 14, 20 y 28.
